# Cyber Code Intelligence for malware detection

[TOC]

# 摘要

​ 不断演变的安卓恶意软件对移动用户构成了严重的安全威胁，基于机器学习 (ML) 的防御技术吸引了积极的研究。由于缺乏知识，许多零日家族的恶意软件可能仍然没有被发现直到分类器获得专门的知识。大多数现有的基于 ML 的方法将需要很长的时间来学习最新的恶意软件家族景观中的新恶意软件家族。现有的基于 ML 的 Android 恶意软件检测和分类方法很难应对恶意软件的快速演变，特别是在零日恶意软件家族的出现和单视图特征的有限代表性方面。

# 目的

​ 在这篇文章中，我们开发了一个新的 **多视图特征智能 (MFI)** 框架，从已知的恶意软件家族中学习目标能力的表示，以识别具有相同能力的未知和不断变化的恶意软件。

​ 新框架执行逆向工程以提取多视图异质特征包括：

  1. 语义字符串特征、

  2. API 调用图特征

  3. smali 操作码（smali opcode）顺序特征。

    • Smali Opcode 是一种用于描述 Android Dalvik 虚拟机指令的语言。Dalvik 虚拟机是 Android 操作系统上运行应用程序的基础。

      Smali Opcode 使用基于文本的语法来描述 Dalvik 字节码指令。它是一种类汇编语言，用于编写和修改已编译的 Android 应用程序的字节码。

      每个 Dalvik 指令都由一个操作码（opcode）和零个或多个操作数组成。操作码表示特定操作的类型，例如加载值到寄存器、进行算术运算或跳转等。操作数提供了指令所需的数据。

      Smali Opcode 在文本文件中按顺序列出指令，并使用特定的语法来定义操作码和操作数。示例如下：

      上述示例中的代码片段演示了一些常见的 Smali Opcode。其中，指令 const/4 将整数常量值（0x1）加载到寄存器 v0 中，指令 const/16 加载整数常量值（0x64）到寄存器 v1 中，指令 add-int 将寄存器 v0 和 v1 中的值相加，并将结果放入寄存器 v2 中，最后的 return 指令将结果从方法中返回。

      通过使用 Smali Opcode，开发人员可以直接修改编译后的应用程序的字节码，以实现一些高级功能、优化性能或进行代码混淆等操作。然而，修改已编译的字节码需要对 Smali Opcode 语法和 Dalvik 指令集有一定的了解。

​ 它可以通过一系列的特征分析、选择、聚合和编码过程从已知的恶意软件家族中学习目标能力的表示，以检测具有共享目标能力的未知 Android 恶意软件。我们创建了一个新的数据集，其中有关于能力的真实信息。在新数据集上进行了许多实验，以评估新方法的性能和有效性。结果表明，在检测具有目标能力的未知 Android 恶意软件时，新方法优于三种最先进的方法，包括:

  1. Drebin
  2. MaaDroid
  3. N-opcode

# 简介

​ Android 在移动操作系统中占主导地位，全球有74.13%的手机运行安卓系统。根据 Statista 的报告，在 2019 年 12 月。仅在官方的 GooglePlay 应用商店中，就有近 300 万个 Android 应用软件。在 2020 年第一季度，尽管从 2018 年初 360 万的峰值下降，但仍有很多应用程序可用 。安卓系统的普及为安卓恶意软件作者提供了激励，而对安卓应用程序进行逆向工程的方法相对容易，导致安卓恶意软件的蓬勃发展。

​ Android 恶意软件家族存在三个主要问题。

  1. 首先，在建立安卓恶意软件家族方面没有统一的标准，因此大多数反病毒公司对家族的命名都不一样。这种临时性的方法为快速分享零日恶意软件的知识造成了障碍。ViruusTotal 通过应用多数投票策略，为特定的恶意软件样本确定一个单一家族的类别标签，从而缓解了命名不一致的问题。
  2. 其次，安卓恶意软件家族的数量在 2018 年超过了 1000 个 ，并且不断激增，没有任何放缓的迹象。由于人类的错误和误判，安卓恶意软件家族的爆炸性增长挑战了从人类专家到基于 ML 的检测器的知识转移。
  3. 第三，安卓恶意软件家族的分布是高度倾斜的，因为在大多数家族中只有少数几个样本可用。例如，在大约 58% 的 Android 恶意软件家族中，只有不到五个样本在 Drebin 数据集中。安卓恶意软件的高度不平衡分布，对特征分析和模式识别的过程提出了挑战。因此，需要另一种替代性的分类方法来检测安卓零日恶意软件。

# 主要贡献

​ 这篇文章的主要贡献有三方面。

  1. 对安卓恶意软件的恶意能力进行了深入的分析，并创建了一个带有验证过的地面真实性分配的数据集，用于检测具有针对性能力的不断发展的安卓恶意软件。（这个名为 "CAPA" 的新数据集将向公众发布为 Android 恶意软件的新研究提供启示。）

  2. 我们提出了一种新的多视图特征智能 (MFI) 方法。MFI 提取多视图异质特征 (源代码视图、API 图形调用视图和 smali 操作码视图。MFI 将这些特征聚集成一个统一的表示，用于描述恶意的能力。此外，MFI 表示可以被送入一个神经模型，以识别具有目标能力的零日恶意软件。

  3. 我们进行了大量的实验，从几个方面评估新的 MIFI 方法，包括识别精度、处理零日系恶意软件的有效性、识别效果和可扩展性。MFI 胜过了三种最先进的方法，包括:

    • Drebin
    • MaMaDroid
    • N-opcode

    在检测具有目标能力的未知安卓恶意软件时。这项工作的所有数据、代码和报告都将在 Github 上发布 (https://github.com/JunTomyang/Android-Malware-Capability-Identification)

    # 新数据和地面实况

    ​ 当把大量的安卓恶意软件与适当的安全 / 隐私相关能力联系起来时，MFI 的直接挑战之一就出现了。据我们所知，在现有的研究数据集中，安全 / 隐私相关的能力没有明确的标签。由于缺乏定义明确的基础真相，现有的公开可用的安卓系统恶意软件数据集需要进一步的标记过程。因此，我们创建了一个带有能力注释的安卓恶意软件数据集，我们将其称为 **CAPA 数据集**。

    ​ CAPA 数据集是通过聚合两个著名的安卓恶意软件数据集 Drebin 和 AMD 得到的。

  • Drebin 数据集包含 5560 个安卓恶意软件应用，来自于 2010 年 8 月至 2012 年 10 月期间收获了 179 个恶意软件家族。

  • AMD 数据集包含 2010 年至 2016 年期间收集的 71 个不同恶意软件家族的 24 553 个 Android 恶意软件样本。

    AMD 数据集比 Drebin 数据集要新，被社区的研究人员广泛使用。

​ CAPA 数据集由 2010 年至 2016 年七年间的 Android 恶意软件样本组成。随着 17 个家族在 Drebin 和 AMD 数据集之间的重复，在最新版本的 CAPA 数据集中，总共包括 72 个家族或 7110 个应用程序。

​ 为了验证我们提出的 MFI 的有效性，作为一个概念证明，我们选择了四种非常常见的能力作为案例研究，它们与四个主要的网络安全问题非常一致: 远程控制、特权、货币化和隐私。不出所料，许多恶意样本拥有这四种能力中的一种或几种：

  1. 僵尸网络攻击 (远程控制)
  2. 未经授权的 root 访问 (权限)
  3. 恶意短信收费 (货币化)
  4. 窃取信息 (隐私)

# 多视图异构结构特征

​ 为了全面描述安卓恶意软件，以便更好地识别与安全隐私有关的能力，我们提取了三个异质特征的视图来代表安卓恶意软件样本。

  1. 源代码中的语义字符串特征:

    ​ 在这项工作中，从 AndroidManifestxml 文件中提取各种类型的语义信息以形成特征。 AndroidManifestxm 文件驻留在所有 Android 应用包的根目录中，并向 Android 系统提供关于应用的基本信息。这四个基本信息来源包括：

    • 应用程序要求的权限
    • 应用程序的软件组件
    • 应用程序要求的硬件组件
    • 过滤的意图

    ​ 这四类信息可以被有效提取，以反映安卓应用的部分特征。具体来说，安卓应用在访问包含敏感信息的资源之前，需要请求终端用户授予某些权限。

  2. 第二个主要的信息来源是从字节码 classes.dex 文件中提取的：

    ​ 对字节码进行反汇编，以检索语义信息，包括应用中使用的 API 调用和数据。四个基本信息源包括：

    • 受限的 API 调用
    • 可疑的 API 调用
    • 使用的权限
    • 网络地址

    ​ 这四种类型的信息提供了字节码的足迹。具体来说，受限的 API 调用表明应用程序的敏感功能；可疑的 API 调用暴露了 Android 恶意软件的安全 / 隐私相关行为；权限信息共同反映了多个 API 调用的关系；网络地址可能与已知的恶意网站或僵尸网络有关。

    ​ 为了从原始恶意软件中获得这些字符串特征，我们使用开源工具 Androguard ( https://github.com/androguard/androguard)。 Androguard 帮助我们从 classes.dex 文件中拆解字节码，并根据上述类别从 AndroidManifest.xml 文件中收集信息。提取的信息形成了语义特征，可以有效地在源代码层面上表示 Android 恶意软件。

    ​ Androguard 是使用 Python 编写的逆向工具，它可以在多个平台上运行 Linux/Windows/OSX。使用它可以反编译 android 应用，也可以用来做 android app 的静态分析（static analysis）

    • androapkinfo.py 查看 apk 文件的包、资源、权限、组件、方法等信息

    • androxml.py 解密 apk 包中的 AndroidManifest.xml 文件

    ​ 在这篇文章中，我们捕获了结构性 API 调用图，以减轻 Android 恶意软件数据集中的规避样本，下图 2 显示了提取 API 调用图特征的过程。API 调用图被提取并嵌入到矢量表示中，旨在捕获恶意软件的行为。

    • 提取 API 调用图
    • 抽象 API 到家庭颗粒度
    • 将 API 调用图嵌入到 向量中

  3. 来自中间的 smali 代码的操作码顺序特征：

    ​ 作为字节码 classes.dex 文件的汇编器 / 反汇编器，smali/baksmali 提供 smali 语言的人类可读代码， smali 代码比 Dalvik 虚拟机用来执行 Android 应用程序的机器代码更容易解释。

    ​ 在本文中，根据 smali 编程规范，字节码 classes.dex 文件被拆解为一组 smali 代码。classes.dex 文件中的每个类都被翻译成一个单独的 smali 文件，每个 smali 文件都包含相应类中的多个方法。每个 smali 条目由个操作码和多个操作数组成，其格式为人类可读的。

    ​ 操作码被分为七个核心指令集以简化分类器的设计, 具体来说，这些指令集是 Move,Return,Goto,If,Get,Put,Invoke。操作码的分离有助于有效地区分相关的操作数，表三总结了指令集的七种类型。

    ​ 随后，获得一个 n-grams（n 元语法）表示，以代表安卓恶意软件。对于一个完整的表示，通过遍历翻译后的 smali 文件来提取操作码序列，以保留 Android 恶意软件的信息，这种安排保证了在 smali 操作码级别的粒度。

# 特征编码和聚合

​ 在这项工作中，多视图的异构特征被预先处理，以形成最终的联合向量表示。为了减少特征维度，去除不相关的特征，缓解下面深度神经网络的复杂性，对于源码中的语义字符串特征，我们根据不同的安全 / 隐私相关能力的特点选择关键的特征子集。例如，对于，恶意短信战能力，将选择SEND SMS、RECEIVE SMS 和WRITE SMS相关的权限特征。

SMS 是Short Messaging Service（短消息服务）的缩写，是一种使用移动设备可以发送和接收文本信息的技术短信内容，以文本、数字或二进制非文本数据为主，目前，这种短消息的长度被限定在 140 字节之内。

​ 对于结构性 API 调用图的特点，如图 2 所示，以提高对 API 变化的弹性并实现可扩展性。因此，所有的 API 调用都被抽象为开发者定义的家族、混清的家族或九个安卓原生家族之一，包括:

​ l) com.google 2) xml 3) apache 4)javax 5) java 6) android 7) dom 8) json 9) dalvik

​ 这些家族是 Android 操作系统的一部分，它们提供了许多不同的 API，可以用于开发 Android 应用程序，这些 API 包括网络、数据库、图形、用户界面和其他功能。

​ 在所有视图的特征向量生成后，我们将这些特征向量汇总，为每个恶意软件应用构建一个联合特征向量。因此，最终的特征向量有 149 个维度。

# 通过神经模型进行能力识别

​ 在这一步，安全 / 隐私相关的能力识别任务被执行，即识别具有信息窃取能力的 Android 恶意软件。最近的一个趋势显示，基于深度神经网络的解决方案在检测安卓恶意软件方面很受欢迎。一旦用高质量的数据进行训练，深度神经网络可以有效地学习安卓恶意软件的复杂特征和行为。在这篇文章中，我们构建了一个深度神经网络架构来进行能力识别任务，同时通过对安卓恶意软件的紧凑表示来实现高的能力识别精度。

​ 对于具体的能力，网络结构在层数和神经元的数量上有微小的变化。优化器从"SGD”、"Adadelta”、"RMSprop 和"Adamax"中选择。而损失函数的选择是 “binary crossentropy”。过度拟合是深度神经网络的主要问题之一，在这项工作中，为了缓解过拟合的问题，我们应用了 dropout 正则化技术。

​ 在我们的深度神经网络的训练阶段，验证集是用训练集的 20% 形成的，以监测训练集和验证集的准确性趋势。验证集表明模型的训练是否正确，是否有过度拟合。如果训练集和验证集的精度增长明显不同，那么学习过程将被终止，并在重新开始训练过程之前修改建模参数以避免过拟合问题。

# MFI 与最先进的方法的比较

​ 现有的工作没有解决李日家族恶意软件的安全隐私相关能力识别问题。我们选择了三个能够进行恶意软件家族分类的基线模型，包括（前文提到）：

  1. Drebin 是第一个用线耳支持向量机分类器进行家庭分类任务的基线方法。
  2. MaMaDroid 是第二种基线方法，采用恶意软件家族模式和随机森林分类器。
  3. N-opcode 是第三种基线方法，N=4，采用随机森林算法。

基线方法是指在机器学习中，使用一些简单的模型作为基准，然后使用更复杂的模型来比较性能。这些简单的模型被称为基线模型，它们可以是随机分类器、最常见类别分类器、简单线性回归和常数预测等。

# 对能力依赖的理解

​ 选择适当的特征来反映特定能力的内在特征，对于准确捕捉不同安全 / 隐私相关能力的模式至关重要。因此，我们进一步确认并确保权限包龄和 API 调用的功能与 Android 开发者文档中的描述一致

  • 僵尸网络攻击能力的特征是根据以下三个标准来选择的：

    • 标准 1：与网络或 WiFiManger 有关的权限和 API 调用功能。
    • 标准 2：与 PushService 或 HttpConnection 相关的服务或 API 调用功能。
    • 标准 3：一些不受信任的网络地域如 URL 功能。
    • 僵尸网络遭网络功击能力的特征数量为 100

  • 具有未经授权的 root 访问能力的恶意软件将削弱安卓设备的安全性并获得特权访问。

    • 恶意软件在获得 root 权限后可以做的事情将没有限制。像 REBOOT、ACCESS _SUPERUSER或exec()等功能将被识别。
    • 如果满足以下标准之一，那么该功能将被选择为未经授权的 root 访问能力。
      • 标准 1：与 System/bin/su 有关的 API 调用特征或 Ljava/Lang/Runime—>exec
      • 标准 2：PackageManager 相关的权限或 API 调用功能，包括在安装软件包、卸载 软件包、更新软件包、删除软件包和重言软件包等方面
      • 标准 3： 与 Bool_Compleled 或 update_device_state 有关的服务和 BroadCast Receiver 特性。
      • 标准 4：与 Receive_Boot_Compleled 相关的意图和行动特征。
    • 未经授权的 root 访问能力的特征数量为 160

  • 恶意软件拥有恶意短信收费能力，可在用户不知情的情况下在后台向高级服务发送短信

    • 所选择的与短信有关的功能包括
      • WRITE SMS 、SEND SMS 、sendTextmessage()
    • 详细的功能选择标准如下。
      • 标准 1：Sms Manager 相关的权限和 API 调用功能，包括send_sms、receive_sms和write_sms。
      • 标准 2：行动和服务功能，包括Data_Sms_Received 和 Smssenderservice
      • 标准 3：BroadCastReceiver 的相关功能，包括SmsReceived和MessageReceived。
    • 恶意短信收费能力的特征数量为 80 个

  • 信息窃取功能将收集用户的感性和隐私信息如联系人名单、位置或设备 ID，然后将其发送到远程服务器。

    • 以下 5 条标准适用于信息窃取能力的过滤特征

      • 标准 1：涉及敏感位置信息的特征 ，包括 access_coarse_location,

        access_fifine_location 和 locationmanager

      • 标准 2：私人信息相关的功能，包括 read_contacts,read_logs, read_history_bookmarks 和 read_phone_state

      • 标准 3：不受信任的网络地址 URL

      • 标准 4：SmsManager 相关权限和 API 调用功能，包括 send_sms、receive_sms 和 write_sms

      • 标准 5：行动和服务特征，包括 Data_Sms_Received 和 SmsSenderService.

    • 信息窃取能力的特征数量为 60。

​ 为了研究能力相关特征选择的有效性，以恶意短信收费能力为例，我们使用 t 分布式随机邻居嵌入 (t-SNE) 算法展示了测试恶意软件的可视化。t-SNE 是一种降维的方法，特别适合数据集的可视化。

​ 图 3 (a) 是使用完整特征集的恶意软件的可视化结果，而图 3 (b) 显示的是基于选定的能力相关特征的结果。红点表示拥有恶意短信收费能力的恶意软件，而绿点表示没有恶意短信收费能力的恶意软件样本。因此，所选择的特征是有辨别力的。

​ Baseline 是使用完整特征集的结果 CDFS 是指使用所选的依赖能力的特征子集的性能

# 单视图与多视图功能

​ 本实验验证了多视图功能是否比单视图功能提供更好的性能，同时确定了零日家庭的安全 / 隐私相关能力。

​ 实验结果列于表六。多视图特征的结果优于单视图特征，在准确率得分和 F1 得分方面分别有大约 1%-4% 和 2%-4% 的提高。

​ 在这项工作中，深度神经网络被设计用来识别恶意软件的安全 / 隐私相关能力。我们研究了识别遭广网各政击能力的深度神经网络的训练和验证损失。训练后的深度神经网络在 300 个 epochs 内迅速收敛。多视图异构特征的直接融合在识别安全 / 隐私相关能力方面的辨别力不够，然而通过使用深度神经网络，可以学习到紧凑的、有鉴别力的表述。这些结果表明，所创建的深度神经网络在聚合多视图异构特征方面是强大的。

# 总结

​ 在这篇文章中，提出了一个新的问题来识别零日安卓恶意软件。为了解决所提出的研究问题，我们设计了一种新的轻量级方法 MFI 来自动识别新检测到的 Android 恶意软件的能力。为了充分捕捉恶意软件的能力特征 MFI，我们使用多视图异构特征来表示每个恶意软件，包括来自源代码的语义字符串特征、结构性 API 调用图特征和 smali 操作码序列特征。我们还设计了一个基于源代码视图的能力相关特征视图。对于每个给定的能力，一个单一的深度神经网络将被训练，以确定测试的恶意软件是否具有特定能力。

​ 在这项工作中，深度神经网络被设计用来识别恶意软件的安全 / 隐私相关能力。多视角异质特征的直接融合在识别安全隐私相关能力方面的区分度不够。通过使用深度神经网络，可以学习到紧凑的、有辨别力的表征。这些结果表明，创建的深度神经网络在聚合多视角异质特征方面是强大的。
